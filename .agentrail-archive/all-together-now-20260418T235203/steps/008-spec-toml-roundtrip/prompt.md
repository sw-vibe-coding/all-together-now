Save-to-TOML currently writes only the flat launch_command; if an agent was created via the New Agent dialog the SpawnSpec structure (transport/host/user/working_dir/agent/...) is lost on save. Add an optional [agent.spec] subtable to AgentEntry (atn-core/src/config.rs). When present on load, the agent is spawned from spec.compose_command() and the spec is registered in state.agent_specs so the UI can show structured fields. save_config writes the spec subtable when the agent has a known SpawnSpec.
