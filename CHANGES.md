# Changes

- Hot-reload keeps `agent_specs` in sync with `agents.toml`. `spawn_config_watcher` previously only updated `agent_configs` + `agent_repo_paths` on reload, so external edits that added or removed `[agent.spec]` subtables left the runtime spec map stale (UI's structured view stayed blank, Save-to-TOML dropped the spec). The watcher now inserts a clone of each entry's spec on add/update, removes stale specs when an entry loses its `spec` field, and drops specs for agents removed from the config entirely.
- Save-to-TOML round-trips the full SpawnSpec. `AgentEntry` gets an optional `[agent.spec]` subtable; when present, `to_agent_config` recomposes the shell command from the spec (so `launch_command` stays blank on disk and can't drift). Startup loader populates `state.agent_specs` from any entries with spec fields; `save_config` writes the spec subtable whenever a spec is known. Two new round-trip tests in atn-core (spec_roundtrips_through_toml, entries_without_spec_omit_the_field_when_serialized); live smoke confirmed that a mosh agent created via the dialog writes a clean spec subtable on Save.
- Agent panel now surfaces the composed launch command: hover the agent name for the full shell line (including `mosh user@host -- tmux new-session -A -s atn-<name> 'cd DIR && AGENT'` for remote agents), plus a small monospace line beneath the header shows a truncated version at a glance. Makes the demo self-documenting ÔÇö no need to hit the API or open the Config editor to see what ATN is actually running.
- Reconnect button in the static dashboard: closes the step-3 UX gap ÔÇö the `/api/agents/{id}/reconnect` endpoint (hard-kill local mosh/ssh + respawn Ôćĺ re-attach to the remote tmux session) is now one click instead of a curl. Button sits next to Restart with its own green styling; click handler closes the SSE stream, POSTs the endpoint, reconnects after 1s, and shows transient "ReconnectingÔÇŽ/Reconnected/Failed" labels. Terminal scrollback is preserved on reconnect (unlike Restart which clears it).
- Docs refresh closes the remote-agent saga: `docs/usage.md` rewritten around empty-start + the New Agent dialog (SpawnSpec fields, transport templates, lifecycle endpoints incl. /reconnect, `ATN_PORT`); new `docs/demo-three-agent.md` walks the CI and live paths; `docs/uber-use-case.md` implementation notes now reflect what actually shipped and cross-link usage/demo/remote-pty; `docs/status.md` adds R1ÔÇôR5 rows; README gets a quickstart that points at the demo. `cargo doc --workspace --no-deps` builds warning-free.
- Three-agent demo: `ATN_PORT` env var (and the machine-readable `atn-server ready on <addr>` stdout marker) let integration tests bind ephemeral ports; three fake-agent shim scripts (`tools/fake-claude`, `tools/fake-codex`, `tools/fake-opencode-glm5`) stand in for the real CLIs; `demos/three-agent/fixtures/*.json` capture the real topology (coordinator/local + two queenbee workers via mosh) and `demos/three-agent/setup.sh` boots ATN (or reuses a running one), POSTs each fixture, and toggles fakeÔćöreal via `ATN_DEMO_REAL=1`; new `crates/atn-server/tests/three_agent_demo.rs` end-to-end test spawns the real binary, POSTs three CI-variant specs, waits for non-`starting` states, routes coordinatorÔćĺworker-hlasm *and* coordinatorÔćĺworker-rpg events, and asserts both reach their inboxes and show up in `/api/events`.
- Remote PTY transport: PTY exit detection ÔÇö `OutputSignal::Disconnected` emitted by the reader on EOF/read-error, picked up by the state tracker to flip `AgentState::Disconnected`; `tools/fake-mosh` records argv to `$ATN_FAKE_MOSH_LOG` and is symlinked as `mosh`/`ssh` for tests; integration tests assert composed argv byte-for-byte for both transports and verify exit-Ôćĺ-Disconnected; new `POST /api/agents/{id}/reconnect` hard-kills the local mosh/ssh child without sending Ctrl-C and respawns (re-attaches to the remote tmux session via `tmux new-session -A -s atn-<name>`); `DELETE /api/agents/{id}` now sends `C-b :kill-session <Enter>` over the PTY for remote agents before shutdown so the remote tmux session is cleaned up; added `docs/remote-pty.md` with the manual queenbee walkthrough.
- New-agent dialog: structured spawn replaces the free-form launch field. `atn-core/spawn_spec::SpawnSpec` captures name/role/transport/host/user/working_dir/project/agent/agent_args, validates per-transport, and composes the shell command (local: `cd DIR && AGENT`; mosh/ssh: `TRANS USER@HOST -- tmux new-session -A -s atn-NAME 'cd DIR && AGENT'`). `POST /api/agents` consumes the new shape, returns the created `AgentInfo` with `spec` + `launch_command`, and 400s with a `missing` field list. Static dashboard and Yew UI both got the matching multi-field dialog with live command preview; Yew gets a real modal replacing the step-1 stub. 13 new unit tests for composition/validation; live end-to-end smoke verified.
- Empty-start: ATN boots cleanly with zero agents (`agents.toml` now empty; seed preserved as `agents.example.toml`); `ProjectConfig.agents` gets `#[serde(default)]`; Yew UI and static dashboard render a dedicated empty state with a "+ New Agent" CTA; new integration test `empty_start_loads_zero_agents`; kicks off a new saga targeting the remote-agent demo (mighty-mike/claude + queenbee/devh1/codex + queenbee/devr1/opencode-z-ai-glm-5); adds `docs/uber-use-case.md`
