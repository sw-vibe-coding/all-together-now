- Ops-polish saga step 6 (watchdog actions — Ctrl-C + blocked_notice + UI badge): closes the ops-polish saga. New `atn-server::watchdog_actor` background task (1 s poll) turns the per-agent stall signal from step 5 into real consequences. `WatchdogState` gains `stall_count_in_run: u32` that increments on every `0→1` stall edge and resets on `on_leaving_running`, so the actor can distinguish "Ctrl-C unstuck it" from "Ctrl-C was ignored and the agent went quiet again" even when the `^C` echo briefly clears the `stalled` flag. Policy: `stall_count == 1` → send Ctrl-C (`InputEvent::RawBytes { bytes: vec![0x03] }`); `stall_count ≥ 2` and still stalled → post a `blocked_notice` `PushEvent` to the agent's outbox (priority High, targeting the coordinator if one's live, else broadcast); when `running_for_secs >= max_running_secs`, escalate once with a separate `blocked_notice` about the running ceiling. Bookkeeping is per-agent and resets when `stall_count_in_run` returns to 0 (i.e. the agent left Running). - Ops-polish saga step 3 (`GET /api/agents/{id}/screenshot`): new HTTP endpoint that returns a rendered terminal snapshot. Query params `format=text|ansi|html` (default `text`), `rows=N` (default 40, clamped 1..=500), `cols=N` (default 120). `text` and `ansi` return `text/plain; charset=utf-8`; `html` returns `text/html; charset=utf-8` with the self-contained `<pre>` from `atn_pty::snapshot::render_html`.
