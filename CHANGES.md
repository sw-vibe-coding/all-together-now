# Changes

- Windowed-UI saga step 4 (carousel layout): third layout mode with a focused primary at ~80% of viewport + prev/next peeks (~9%) on either side, plus `[◀]` / `[▶]` cycle buttons in the top bar. `computeLayout` gains a carousel branch that filters out manually-minimized agents (they stay in the bottom dock, deliverable 5) and builds a name-ordered ring: focus defaults to `selectedId` → coordinator → first, prev/next wrap around the ends, and n≤2 collapses to just the primary or primary+one peek. `setLayoutMode` now accepts `'carousel'` (leaves stack mode when coming from stack, clears any lingering maximized state so carousel/stack math isn't short-circuited). New `carouselCycle(delta)` moves `selectedId` through the eligible ring; `.carousel-btn[data-carousel]` buttons wire prev/next. `windowSelect` re-runs the layout when carousel is active so clicking a header snaps that window to the primary slot. Compact-tile `swap-to-focus` handler also updates `selectedId` so clicking a peek (rendered compact at 9% width) promotes it. Dock-cell clicks in carousel restore + focus. `#carousel-nav` toggles via `updateCarouselNav`. Live browser smoke (3-agent demo): Tiled → Carousel lays out coord primary (1126px of 1400), worker-rpg left peek (125px), worker-hlasm right peek (125px); `◀/▶` cycle shuffles name-order ring; peek click jumps to center; manually minimized worker-hlasm goes to dock and is skipped by cycle; dock-click restores + focuses; Tiled toggle back restores 3-tile grid; persistence round-trips. cargo test + clippy clean.
- Windowed-UI saga step 3 (stack layout): new `Tiled` / `Stack` toggle in the top bar (`.layout-selector`). Switching to Stack reconciles per-window state via `enterStackMode` — one primary (state: `normal`, derived from `selectedId` → coordinator role → first by sort) and everyone else dock-minimized; `computeLayout` short-circuits on `layoutMode === 'stack'` to place the primary centered at ~80% × 90% of viewport and emit the rest as implicit dock entries. Clicking a dock cell in Stack mode routes to new `stackPromote` (swap which window is normal vs minimized + select the clicked); dock-cell clicks in Tiled still restore as before. Switching back to Tiled runs `leaveStackMode` and restores every non-pinned window to `normal`. New agents added while Stack is active auto-join minimized via `addAgentPanel`. Startup re-runs `enterStackMode` if the persisted `layoutMode` is `stack` so refreshes render consistently. Helpers: `pickStackPrimary`, `setLayoutMode`, `updateLayoutSelector`, `setupLayoutSelector`. Live browser smoke (3-agent demo): Tiled → Stack yields 1 normal / 2 minimized with coord primary; dock-click swaps primary to worker-hlasm; Tiled restores 3 normals; persistence round-trips via `atn-scale-ui-v1`. cargo test + clippy clean.
- Windowed-UI saga step 2 (window chrome): each panel header now carries six icon buttons (`_` minimize / `□` maximize-restore toggle / `📌` pin / `▸` config / `↻` reconnect / `✕` delete). `layoutState.windows[id] = { state: 'normal'|'minimized'|'maximized', pinned }` tracks per-window state (plus `layoutState.selectedId` for click-select). Clicking a header outlines that window in accent green (the "selected" window — where step-5 bare-key shortcuts will act). Clicking a chrome icon fires its action without selecting. Minimized windows move off-screen; a new `#window-dock` strip along the bottom of the dashboard renders a 180×48 cell per minimized agent (role + name + state badge + last-line). Clicking a dock cell restores that agent. Maximize lifts one agent to ~80% of viewport centered; all others are implicitly dock-minimized for the duration (their stored state stays unchanged). `windowSelect` / `windowMinimize` / `windowMaximizeToggle` / `windowRestore` plus `reconnectAgent` + `confirmDelete` handlers wire the chrome to the state map. Dock rendering lives in `renderWindowDock`; `escapeHTML` helper used throughout. 60 workspace tests still green; clippy clean.
- Windowed-UI saga step 1 (tiled foundation): replaced the heat-driven treemap with a simple, stable tiled layout. New `computeLayout` places the coordinator in a prominent left slice (55% of viewport width) and grids the remaining workers in name order at equal size on the right. No heat-based sizing, no squarify sort-flipping, no scale hysteresis — Refresh is genuinely idempotent. `applyLayout` now takes a `{windows: [...]}` shape (was `{focus, tiles, groups}`). `buildLayoutItems` sorts by name. `pickTermScale` drops the `currentScale` hysteresis param (no longer needed once tile sizes are stable). `layoutState` gains `layoutMode: 'tiled'` and `sortMode: 'name'` fields (persisted in `atn-scale-ui-v1`; later saga steps introduce stack + carousel + recent-sort). Dead-but-unused treemap code (`squarify`, `groupedSquarify`, `autoPickFocus`, `FOCUS_WIDTHS`) is left in place for later cleanup in step 6.
- Dialog polish for the coordinator-once / workers-many flow. When `transport=local`, the user/@/host group is **hidden** (not just disabled) in both the New Agent dialog and the per-agent Config editor's spec mode — the coordinator create path shows just name + role + transport + dir + project + agent + args. `readSpawnSpec` and `readSpecFromEditor` force `user: null, host: null` for local transport so hidden-input residue can't leak into the composed command. New localStorage key `atn-last-host` remembers the host after any successful POST /api/agents with mosh/ssh transport; when the next dialog open flips transport to non-local, the host input is pre-filled from that value (user, dir, agent stay blank — they differ per worker). + New Agent button now focuses the name input so the flow is keyboard-first.
- Config editor round-trips SpawnSpec. `UpdateAgentBody` on `PUT /api/agents/{id}` gains an optional `spec: Option<SpawnSpec>`. When present, the server validates (400 with missing-field list on fail), recomposes `launch_command` from `spec.compose_command()`, replaces `state.agent_specs[id]`, derives `repo_path` (local → working_dir, remote → base_dir) and role, and runs the usual shutdown + respawn. When absent, the legacy flat overlay (`name/repo_path/role/setup_commands/launch_command`) is unchanged and the stored spec is preserved. The per-agent Config editor in the static dashboard now detects `agent.spec` in the list response and renders a structured editor (role/transport/user/host/working-dir/project/agent/agent_args + live composed-command preview + missing-fields inline error); legacy TOML-loaded agents continue to show the flat editor. `applyConfig` dispatches the right body shape for each mode. Live smoke: mosh agent → PUT `{spec:{...}}` changing working_dir/agent/agent_args → GET shows the new spec + matching composed launch; PUT `{launch_command:...}` legacy shape overrides launch without touching the stored spec; PUT invalid spec returns 400 with `missing: host, user`. No Rust tests added; 60 workspace tests still green; clippy --all-targets clean.

- Scale-UI step 7: search, filter chips, group-by, saved layouts. Filter-bar visibility tied to agent count (hidden on empty start). 60 workspace tests green; clippy --all-targets clean.

- Scale-UI step 5: pin row, keyboard shortcuts, persistence. New `.pin-row` strip above the dashboard renders mini compact-tiles for user-pinned agents (fixed 180×82, sparkline + last-line + state badge, × button to unpin). Pinned agents are excluded from the heat-sized treemap and from focus auto-pick; pinning the current focus returns focus to auto-pick. `.btn-pin` pin button on every agent-header (clicking sets `.pinned` style). Keyboard: `1–9` focus Nth hottest unpinned, `0` focus coordinator, `f` toggles focus panel between medium (40–50%) and large (55–70%) widths, `p` pin/unpin current focus, `/` focuses filter input (stub until step 6), `Esc` clears focus → auto-pick. `isTypingTarget` guard ignores keys inside inputs/textareas/selects/xterm. localStorage key `atn-scale-ui-v1` persists `{ pins, focusId, focusSize }`; hydrated at init before the first render. Existing `.agent-panel` click listener now honors `[data-no-focus]` so clicking the pin button doesn't also swap focus. 60 workspace tests green; clippy --all-targets clean.
- Treemap layout: `.dashboard.treemap-mode` replaces the cols-1..cols-4 grid with an absolutely-positioned canvas. New `squarify(rect, items)` classic Bruls/Huijsen/van Wijk squarified treemap (with an on-load self-test covering area-fill, bounds, single-item, empty). `layoutState` carries `focusId` + `freezeUntil` + a per-agent smoothed-heat EWMA (~5s half-life at 1 Hz; keeps tiles from jittering on byte bursts). `computeLayout` carves out a focus panel (40–55% of viewport width, scales with relative heat) and runs squarify on the rest. `applyLayout` places panels via absolute `left/top/width/height`, flips each tile between xterm and compact mode based on `pickTermScale`, and follows with `updateTileScales`. `swap-to-focus` CustomEvent is now handled end-to-end: sets `focusId`, freezes layout for 5s, calls `setFocus` for the PTY+xterm fit. Clicks on `.agent-header` and compact tiles both dispatch it. Refresh tick runs every 1.5s unless frozen. Auto-pick prefers a coordinator; falls back to highest-heat overall. 60 workspace tests green; clippy clean.
- CSS-scaled xterm: every panel nests `.agent-terminal > .term-scale > xterm-host`. xterm is created at fixed native geometry (`TERM_NATIVE_COLS=120`, `TERM_NATIVE_ROWS=40`); tile size tracking happens via `transform: scale(var(--term-k, 1))` on the wrapper. `pickTermScale(tileW, tileH, natW, natH)` snaps to `{0.25, 0.33, 0.5, 0.66, 0.8, 1.0}` or returns null below threshold (→ compact tile). `updateTileScales` iterates every xterm panel on window resize / ResizeObserver / grid-class change and updates the CSS var — no `fitAddon.fit()`, no `POST /api/agents/{id}/resize`. Only `setFocus(id)` (dev-exposed as `window.setFocus`) still fits + syncs PTY size, for the single focused tile. Inline self-test (`testPickTermScale`) runs on load and logs PASS/FAIL for 8 cases.
- Compact tile: small-footprint agent tile variant with no xterm inside — header (name/role/state badge) + body (inline SVG sparkline, truncated last-line, click-to-focus hint). SSE output pipeline now also feeds a per-agent last-line buffer (ANSI-scrubbed, ≤200 chars); a 1Hz poll of `/api/agents/heat` fills a 60-sample `bytes_per_sec` ring per agent that the sparkline reads. Clicking a compact tile dispatches a `swap-to-focus` CustomEvent (step 4 will wire the layout side; the default handler just logs). `window.toggleCompactMode(id)` from devtools swaps between xterm and compact for manual testing before the step-4 auto-pick. Supporting helpers: `ensureAgentMeta`, `recordOutputBytes`, `pollHeat`/`startHeatPolling`, `sparklinePath`, `createCompactTile`, `refreshCompactIfPresent`. 60 workspace tests still green; clippy `--all-targets` clean.
- Heat score: new `atn-server/src/heat.rs` tracks a per-agent EWMA of output bytes/sec (time-aware; long quiet gaps decay properly) plus state multipliers (awaiting_input / error / blocked get boosted, disconnected gets muted) and a floor so cold agents keep a visible tile. `spawn_heat_tracker` subscribes to the PTY output broadcast, feeds byte counts into a shared `HeatMap`, and cleans up on `Disconnected` / channel close. Wired into every spawn path (startup loader, create, restart, reconnect, hot-reload); removed on delete and hot-reload removal via a small `rearm_heat_tracker` helper. `GET /api/agents/heat` decays each entry to the current instant before returning `{id, heat, bytes_per_sec, state_boost}` per live agent. 6 unit tests cover EWMA bursts/decay, state-boost layering, floor, clamping. Live smoke verified heat rises on spawn, decays over silence, climbs again when inputs pump output, and drops on delete.
- Hot-reload keeps `agent_specs` in sync with `agents.toml`. `spawn_config_watcher` previously only updated `agent_configs` + `agent_repo_paths` on reload, so external edits that added or removed `[agent.spec]` subtables left the runtime spec map stale (UI's structured view stayed blank, Save-to-TOML dropped the spec). The watcher now inserts a clone of each entry's spec on add/update, removes stale specs when an entry loses its `spec` field, and drops specs for agents removed from the config entirely.
- Save-to-TOML round-trips the full SpawnSpec. `AgentEntry` gets an optional `[agent.spec]` subtable; when present, `to_agent_config` recomposes the shell command from the spec (so `launch_command` stays blank on disk and can't drift). Startup loader populates `state.agent_specs` from any entries with spec fields; `save_config` writes the spec subtable whenever a spec is known. Two new round-trip tests in atn-core (spec_roundtrips_through_toml, entries_without_spec_omit_the_field_when_serialized); live smoke confirmed that a mosh agent created via the dialog writes a clean spec subtable on Save.
